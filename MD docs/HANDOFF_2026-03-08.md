## Feature: Critical Path

### Show Critical Path (🔴)
- Finds latest Gate milestone (highest due date with `milestone=true` and "gate" in name)
- Walks backward through `dependencies` table collecting ALL ancestors (not just longest chain)
- For Gate milestones with no explicit deps: falls back to all non-gate tasks in same phase
- On activate: saves `_savedCritical` snapshot, sets `task.critical=true` for all path tasks
- On deactivate: restores `_savedCritical`
- Respects `manualUncritical` — tasks user explicitly unchecked are excluded

### Longest Chain (🔗)
- Memoized backward walk finding the single longest sequential path by calendar days
- Same Gate milestone fallback logic as above
- Shows chain length and total duration in notification

### Manual Critical Flag
- User checks/unchecks `#taskCritical` in Edit modal
- On save: `setManualUncritical(id, !critical)` writes to `localStorage` key `manualUncritical_<PROJECT_NAME>`
- On page load: `getManualUncritical()` reads localStorage; task mapped with `manualUncritical: true/false`
- `autoMarkCriticalByGateDeadline()` skips tasks where `task.manualUncritical=true` (preserved but currently auto-marking is disabled anyway)
- When critical path or longest chain activates: `getManualUncritical().forEach(id => set.delete(id))` excludes those tasks
- localStorage persists across refreshes; cleared only when user re-checks critical and saves

### Bar Color Logic
- Red bar: `task.critical === true` in memory → CSS class `.gantt-bar.critical`
- Yellow row background: `task.exceedsGateDeadline === true` (gate deadline exceeded)
- The `#filterCritical` checkbox is **hidden** in UI but still used internally by critical path / longest chain toggles

## Feature: Dependency System

### Auto-Detection (Import)
- `server.py` ~line 200: finish-to-start with 0–2 day delta
- `0 <= B.start_date - A.end_date <= 2 days` → creates dependency A→B
- Applied only on Excel import, not retroactively

### Dependency Picker (Edit Modal)
- Shows tasks ending ≤ task's start date + 2 days, grouped by phase
- CSS classes: `.dep-phase-group`, `.dep-phase-header`, `.dep-phase-tasks`
- Filter search queries these classes

### DB Table: `task_dependencies`
```sql
id, project_name, predecessor_id, successor_id
```

## Feature: Gate Deadline System

### Gate Milestones
- Detected by: `task.milestone === true AND task.name matches /gate/i`
- Two Gate 5 milestones: id=79 (Saturn, ends 2028-03-25), id=141 (F31, ends 2028-01-30)
- `excel_parser.py` sets `'milestone': 1` for names matching `^Gate\s*\d+\s*$`

### Gate Baselines (Change Log)
- `gate_baselines` table: stores original gate dates on first import
- `gate_change_log` table: records every gate date change with reason
- `📋 Gate Change Log` button shows full history

### Cascade Date Change
- When editing a task date: `checkAndShowCascadePreview()` shows:
  1. All downstream tasks that would shift (finish-to-start deps, ≤2 day delta)
  2. Any gate milestones affected by the shifts
- User confirms or cancels before any save occurs
- Gate phase guard: only same-phase gates are checked (prevents cross-phase false positives)

## Feature: Tailed-Out Tasks
- `task.tailed_out = true` → task is cancelled/removed from scope
- Shows 🚫 icon, greyed out row
- Excluded from critical path and dependency calculations

## Known Behaviors / Gotchas

1. **`task.critical` is NOT auto-set from DB** — `autoMarkCriticalByGateDeadline()` used to auto-set it but that was removed (caused user's manual unchecks to revert). Now `task.critical` only becomes `true` if: (a) user checks it in Edit modal, or (b) Critical Path / Longest Chain overlay is active.

2. **All tasks currently have `critical=0` in DB** — this is correct. Critical is a session/display concept now, not persisted unless user explicitly checks and saves.

3. **`manualUncritical` is localStorage only** — not in DB. Per-project key. If user clears browser storage, exclusions are lost.

4. **Gate 5 has no explicit dep links** — the 2-day auto-detection gap is too large (30–60 days). The critical path algorithm handles this with the "same-phase fallback" (uses all non-gate tasks in Gate 5 phase as implicit predecessors).

5. **`filterCritical` checkbox is hidden** — still in DOM, still functional. The Critical Path and Longest Chain buttons check/uncheck it programmatically to trigger the `applyFilters()` filter logic.

6. **Route pattern**: `/project/<project_name>` (NOT `/dashboard/<name>`)

7. **Brace balance check** — whenever editing `dashboard.html`, verify with:
   ```python
   python -c "
   import re
   with open('templates/dashboard.html', encoding='utf-8') as f: s = re.findall(r'<script(?![^>]*src)[^>]*>([\s\S]*?)</script>', f.read())[0]
   print(s.count('{'), s.count('}'), s.count('{')-s.count('}'))
   "
   ```
